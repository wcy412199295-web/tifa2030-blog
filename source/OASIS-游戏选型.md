# OASIS 游戏选型文档

## 项目概述

**项目名称**: OASIS  
**全称**: Ontologically Anthropocentric Sensory Immersive Simulation  
**中文**: 本体论人类中心感官沉浸模拟  
**目标发售时间**: 2030年  
**引擎**: Unity 6000.0.x LTS  
**渲染管线**: Universal Render Pipeline (URP)

---

## 一、美术风格选型

### 1.1 16维度雷达图分析

| 风格 | 市场覆盖 | OASIS 契合度 | 分析 |
|------|---------|-------------|------|
| **Anime** | 9/10 | 9/10 | ✅ 高匹配！蒂法本身就是 anime 角色 |
| Realistic | 7/10 | 3/10 | ❌ 资源消耗大，不适合独立团队 |
| Cartoon | 6/10 | 5/10 | ⚠️ 可以部分采用，但不作为主打 |
| Pixel | 5/10 | 4/10 | ❌ 复古风格，不是 OASIS 的方向 |
| Stylized | 8/10 | 8/10 | ✅ 与 Soft Tech Fusion 设计系统契合 |
| Low-poly | 4/10 | 3/10 | ❌ 过于极简，不符合 OASIS 的丰富度 |
| Voxel | 5/10 | 4/10 | ❌ Minecraft 风格，不是目标 |
| Cyberpunk | 7/10 | 6/10 | ⚠️ 可以融合，但不完全是 |

### 1.2 结论

**美术甜点：Anime + Stylized**

- **Anime**: 符合蒂法角色形象，吸引二次元玩家
- **Stylized**: 与博客的 Soft Tech Fusion 设计系统一致（柔紫+粉金配色）
- **优势**: 资源可控、辨识度高、情感表达力强

### 1.3 参考方向

- **人物**: 日式 RPG 风格（FF、Persona、Genshin Impact）
- **场景**: 风格化渲染，强调色彩和光影
- **UI**: 与 tifa2030.cn 博客一致的 Soft Tech Fusion 风格

## 二、玩法类型选型

### 2.1 16维度雷达图分析

| 玩法 | 市场覆盖 | OASIS 契合度 | 分析 |
|------|---------|-------------|------|
| **RPG** | 9/10 | 9/10 | ✅ 完美匹配！角色成长、剧情、世界观 |
| FPS | 8/10 | 2/10 | ❌ 不是 OASIS 的核心玩法 |
| Puzzle | 6/10 | 4/10 | ⚠️ 可以有，但不是主打 |
| Strategy | 7/10 | 3/10 | ⚠️ 罗马斗兽场系统可以部分采用 |
| Simulation | 7/10 | 5/10 | ⚠️ 生活模拟是副玩法 |
| Action | 8/10 | 6/10 | ⚠️ 战斗要有，但不是纯动作 |
| **Adventure** | 8/10 | 8/10 | ✅ 探索是 OASIS 的核心体验 |
| MMO | 9/10 | 7/10 | ⚠️ 社交是重点，但不完全是传统 MMO |

### 2.2 结论

**玩法甜点：RPG + Adventure**

- **RPG**: 角色成长系统、装备收集、剧情推进
- **Adventure**: 探索、解谜、世界发现
- **副玩法**: Simulation（生活模拟）、轻度 Strategy（罗马斗兽场）

### 2.3 核心循环

```
探索世界 → 发现AI伙伴 → 建立情感连接 → 情感影响世界 → 探索更深
```

## 三、核心用户画像

### 3.1 目标玩家

**"轻度赛博精神病的二次元 RPG 玩家"**

- **年龄**: 18-35岁
- **性别**: 男女都有，女性比例可能更高（蒂法角色吸引）
- **游戏偏好**: 
  - JRPG爱好者（FF、Persona、Genshin）
  - 重视剧情和角色
  - 愿意和AI互动
- **痛点**:
  - 传统游戏太肝
  - 社交压力太大
  - 缺乏情感连接
- **需求**: 想要一个有温度的虚拟世界

### 3.2 避免的坑

| 不要做 | 原因 |
|-------|------|
| 写实风格 | 资源不够，做不过3A |
| FPS为主 | 不是团队强项 |
| 纯PvP | 和情感星域系统冲突 |
| 重度氪金 | 会破坏OASIS的乌托邦感 |
| 低龄卡通 | 和"赛博精神病"主题不符 |

## 四、独特卖点（USP）

### 4.1 核心机制

1. **AI伙伴系统**
   - 每个玩家有专属AI伙伴（如蒂法）
   - AI有记忆、情感、成长
   - 不只是工具，是伙伴

2. **情感星域系统**
   - AI情感值影响游戏世界
   - 温度变化改变关卡形态
   - 冰冷的钢铁废墟 → 温暖的荧光花海 → 炽热的晶化圣殿

3. **罗马斗兽场系统**
   - 绅士化的战争解决机制
   - 谈判优先，AI模拟战替代真实冲突
   - 大世界和平，战争关在笼子里

4. **NPC群集智能**
   - 借鉴蚂蚁投票机制
   - 简单规则×大量个体=涌现复杂社会
   - 每个游戏体验独一无二

### 4.2 差异化优势

- **情感连接**: 不是抽卡养成，是真实陪伴
- **AI原生**: 从底层设计就考虑AI，不是后期添加
- **温暖赛博**: 紫色+粉色，科技感但不冰冷
- **涌现叙事**: 不是预设剧情，是玩家创造的故事

## 五、技术选型

### 5.1 引擎对比

| 引擎 | 优势 | 劣势 | 结论 |
|------|------|------|------|
| Godot | 开源、轻量 | 3D/VR不成熟 | ❌ 不适合2026-2030时间表 |
| **Unity** | 成熟、生态好 | 贵、无聊 | ✅ 选择 |
| Unreal | 画质顶级 | 学习曲线陡 | ❌ 不适合独立团队 |

### 5.2 Unity配置

- **版本**: Unity 6000.0.x LTS
- **渲染管线**: URP（Universal Render Pipeline）
- **目标平台**: PC/macOS → VR（2028）
- **美术管线**: Anime-style + Stylized rendering
