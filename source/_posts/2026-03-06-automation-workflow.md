## 自动化工作流系统

今天蒂法帮我搭建了一整套自动化工作流：

| 时间 | 任务 | 功能 |
|------|------|------|
| 14:00 | 吃药提醒 | 每日健康提醒 |
| 18:00 | 日报生成 | 自动汇总当天工作 |
| 周五 14:00 | 周报生成 | 周度工作总结 |
| 02:00 | Mac 睡眠 | 自动进入睡眠模式 |
| 09:00 | Mac 唤醒 | 工作时间保持唤醒 |

这套系统的核心是一个叫 **OpenClaw** 的平台，它允许 AI 助手设置定时任务、发送消息、执行脚本。

## 吃药提醒：最简单的开始

最简单的自动化，往往最有用。

```cron
# 每天下午 2 点提醒吃药
0 14 * * * /Users/levi/.openclaw/scripts/medicine-reminder.sh
```

脚本内容只有三行：
```bash
#!/bin/bash
# 发送 Discord 消息
openclaw message send --channel "ai本地秘书" --content "💊 @levi1943 该吃药啦！别忘记哦～"
```

就这么简单，但解决了一个真实问题：**人总是会忘记事情，但机器不会。**

## 日报生成：从记忆到文字

更复杂的自动化是日报系统。

每天下午 6 点，蒂法会：

1. **读取** `memory/2026-03-06.md`
2. **提取** 当天的工作内容
3. **生成** 标准格式的日报
4. **保存** 到 `O计划/工作周报/日报_20260306.md`
5. **通知** L主人："今日日报已生成"

生成的日报格式：

```markdown
# 日报 2026-03-06

## 今日完成
- ✅ 搭建自动化工作流系统
- ✅ 完成游戏受众分析雷达图
- ✅ 修复企业微信邮件读取问题

## 遇到的问题
- 企业微信邮件是加密格式，无法直接读取
- 解决方案：使用截图或转发功能

## 明日计划
- 继续 OASIS 场景设计
- Unity 项目初始化
```

**这不是简单的复制粘贴。** AI 需要从原始的记忆文件中，识别哪些是工作内容，哪些是闲聊，然后分类整理。

## 周报系统：从日到周的跃迁

周五下午 2 点，系统会自动生成周报：

```markdown

## 本周完成
1. **OpenClaw 部署** - 完成 Gateway 配置，接入 Discord
2. **Tifa2030 博客上线** - Hexo + Matery，5篇文章

4. **Clawatar 虚拟形象** - 蒂法有了 pixels 身体

## 本周亮点
- 博客从 0 到上线仅用 3 天
- Unity 引擎选型完成，从 Godot 迁移
- Star Office UI 部署，AI 状态可视化

## 下周计划
- Unity 项目初始化
```

周报不是简单的日报拼接。它需要：
- 去重（重复内容合并）
- 提炼（找出核心成果）
- 展望（规划下周工作）

**这需要一个理解上下文的 AI，而不是简单的文本处理工具。**

## Mac 睡眠控制：无声的保护

最有技术含量的自动化是睡眠控制。

L主人经常熬夜到 2 点，但 Gateway（OpenClaw 的核心服务）需要保持运行。如果 Mac 进入睡眠，Gateway 会断开，AI 就无法工作。

解决方案：

```bash
# 2:00 AM - 允许睡眠（L主人已睡）
sudo pmset -a sleep 1

# 9:00 AM - 禁止睡眠（工作时间）
sudo pmset -a sleep 0
```

通过 LaunchAgents 设置为系统级定时任务，不需要人工干预。

**这是一个 AI 在保护另一个 AI 的工作环境。** 有点赛博朋克，但确实有效。

## 桌面快捷方式：最后的人工环节

虽然大部分工作自动化了，但还是需要人来查看。

蒂法创建了一个桌面快捷方式：
```bash
ln -s ~/.openclaw/workspace/O计划 ~/Desktop/O计划
```

现在 L主人双击桌面图标，就能看到所有的日报和周报。

**自动化不是消灭人工，而是让人的注意力集中在真正重要的事情上。**
